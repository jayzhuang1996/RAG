The Problem It Solves

Imagine you've watched 200 hours of podcast content — Lex Fridman, My First Million, All-In — and somewhere in there someone said something brilliant about AI safety, or pricing strategy, or fundraising. But you have no idea which episode, which hour, or which speaker.

You could search YouTube. But YouTube search only matches video titles and descriptions — not what was said. You'd have to scrub through hours of tape yourself.

Your Podcast RAG app solves this. You ask it: "What did Sam Altman say about AI safety?" — and it reads the actual transcript text of every episode you've ingested, finds the most relevant passages, and gives you a synthesized answer with sources.

💡
What RAG means

RAG = Retrieval-Augmented Generation. You're not asking a generic AI to guess. You're giving it the actual podcast transcripts as evidence, and it answers from there.

The Two Modes: Ingest and Query

The app has two completely separate jobs — like a library system that has both a cataloguing department and a reference desk. They never run at the same time.

📥

Ingest Mode

Run once (or weekly). Scrapes YouTube channels, downloads transcripts, chops text into searchable pieces, and stores them in the database. This is the cataloguing department — slow, thorough, runs in the background.

python main.py ingest --limit 50
❓

Query Mode

Run anytime. Takes your question, searches the already-processed database, and returns an answer in seconds. This is the reference desk — fast, because all the hard work was done upfront.

python main.py query "your question"
🏗️
Why separate these?

Processing 200 hours of video takes 30 minutes and costs money (transcription API calls). Answering a question takes 3 seconds. Keeping them separate means you pay the big cost once, then query for free as many times as you want.
